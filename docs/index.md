# MulmoTerminal Guide

**A terminal-first, modern development environment for the age of AI coding agents.**

One engineer, many agents. MulmoTerminal lets you run several AI coding agents
([Claude Code](https://claude.com/claude-code) and OpenAI's **Codex**) **in parallel**, see **what each one
is doing and where**, and step in only where you're needed — all on top of a real terminal. Around that it
adds **visibility**, **automation**, and **error investigation**, and makes almost everything **extensible
through a small DSL**, so it bends to *any* developer's workflow.

**AI コードエージェント時代の、ターミナルを基軸にした現代の開発環境。** 1 人の開発者が複数のエージェント
（Claude Code / Codex）を**並行して走らせ**、それぞれが**どこで何をしているか**をひと目で把握し、
**手が要るものにだけ**入る。その上に**可視化・自動化・エラー調査**を自然に載せ、機能も設定も**小さな DSL で拡張**できるので、
**どんな開発者のどんなワークフローにも**合わせられます。

![A board of parallel AI-agent terminals](guide/images/grid-2x2.png)

## The four pillars / 4 本柱

1. **監督 (Supervise)** — a **cockpit for parallel agents**. Dispatch many; triage by status; attend only to the ones that need you.
2. **可視化 (See)** — each agent's status, model, context, git state, tool-call **timeline**, and estimated cost, at a glance.
3. **自動化 & 調査 (Automate & investigate)** — run scripts in a cell, and turn a wall of failing output into a short **AI diagnosis**.
4. **拡張 (Extend)** — a **DSL** for header buttons, chips, launchers, and per-project config — shape the terminal to fit you.
