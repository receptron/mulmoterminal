# MulmoTerminal ガイド（日本語）

## コンセプト — なぜこのターミナルか

AI コードエージェント（**Claude Code** / **Codex**）は、1 つのタスクに数分かかります。1 体を見張っていると、
その間あなたの手は空きっぱなし。ボトルネックは CPU でも端末でもなく、**あなたの注意**です。

MulmoTerminal はそこを起点に生まれました。まず——

> **複数のエージェントを並行して走らせ、状態で triage し、手が要るものにだけ入る「司令塔」。**

そしてそこから進化して、単なる並列端末ではなく、**ターミナルを基軸にした現代の開発環境**になっています。

### 4 本柱

1. **監督** — グリッドは**エージェント群のコックピット**。N 個のタスク／worktree に N 体を放ち、
   *作業中（青）／要対応（琥珀）* の色と通知で、**呼ばれたセルにだけ**行く。
2. **可視化** — 各エージェントの **状態・モデル・コンテキスト量・git・ツール呼び出しタイムライン・推定コスト**を一目で。
   「どこで何をしているか」が常に見える。
3. **自動化 & エラー調査** — スクリプトをセル内で走らせ、失敗したら大量のログを **AI が短く診断**（原因・直し方）。
4. **拡張（DSL）** — ヘッダーのボタン／チップ、ランチャ、プロジェクト設定を**小さな DSL で拡張**。
   端末を**自分のワークフローに合わせて**成形できる。だから、どんな開発者にも合う。
