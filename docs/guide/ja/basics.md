## 単一ビューとグリッドビュー

MulmoTerminal には 2 つの表示モードがあります。上部ツールバーの **チャット / グリッド** アイコンで切り替えます。

- **単一ビュー**：1 つのセッションに集中する画面。左にチャット風の履歴、右に GUI パネル（図・フォーム・画像・書類）。
- **グリッドビュー**：複数のターミナルをタイル状に並べて**同時に**動かす画面。このガイドの主役です。

![単一ビュー](../images/single-view.png)

> グリッドの各セルは独立した PTY です。1 つのセルで Claude が考えている間に、別のセルでシェルを叩いたり、
> 別プロジェクトの Codex を動かしたり、テストを走らせたりできます。

## ターミナルを起動する（ランチャフォーム）

グリッドの空きセルには **ランチャフォーム**が表示されます。ここから起動方法を選びます。

![空きセルのランチャフォーム](../images/grid-launch-form.png)

| 部分 | 役割 |
|---|---|
| **Claude / Codex** トグル | このセルで動かすエージェントを選ぶ |
| **WORKING DIRECTORY** | 作業ディレクトリを入力（`▶` で起動）。よく使うディレクトリは設定の *cwd presets* から補完 |
| **OR ISOLATE IN A WORKTREE** | git リポなら、タスク名を入れて **＋ New worktree**。作業を隔離した git worktree を作って起動 |
| **OR LAUNCH** | Claude 以外の**起動コマンド**（例：`Shell`、`Node REPL`）。設定で登録したものが並ぶ |

## ターミナルセルの見方

起動するとセルはターミナルになります。ヘッダーは 2 段です。

![起動したセル（2段ヘッダー）](../images/grid-one-cell.png)

- **1 段目（情報）**：状態ドット・ディレクトリ・git チップ（`⎇ ブランチ ●変更数`）・モデル/コンテキスト量・そのセッションが今やっていること・拡大/閉じる。
- **2 段目（操作）**：接続状態（`connected`）・添付・ファイルブラウザ・GitHub・タイムライン 🕘 など。

> 状態は**色**でも分かります。青みがかった枠＝**作業中**（エージェントが考えている）、
> 琥珀色＝**要対応**（入力待ち／見ていない出力あり）、通常＝アイドル。

## 複数セルとページ

- ツールバーの **New terminal（＋）** で空きセルを追加。セルは 1 ページ **9 個**まで。あふれると次のページ（タブ）になります。
- **Toggle grid cell ordering** で並べ替えモードに入ると、各セルの `◀ ▶` で位置を入れ替えられます。

![2×2 の並行ターミナル](../images/grid-2x2.png)

## 1 つを拡大する（フィルムストリップ）

セルの **⤢**（拡大）を押すと、そのセルが大きく表示され、他のセルは下部の**フィルムストリップ**にサムネイルとして並びます。
サムネイルのヘッダー余白をクリックすると、その端末に**切り替え**られます。**⤡** で元のグリッドに戻ります。

![拡大（フィルムストリップ）](../images/grid-zoom.png)

## Claude と Codex

同じグリッドの中で、セルごとに **Claude** でも **Codex** でも起動できます。どちらも同じターミナル体験・永続化・GUI パネルの仕組みを共有します。
起動時のトグルで選ぶだけです。
