# feat #595 — ターミナル間の自動往復（PR 1: 1往復）

対象 issue: https://github.com/receptron/mulmoterminal/issues/595
先行: #566（受け渡し）/ #574（pull 方向）/ #582（codex のターン境界検知）

Phase 3 のうち **1往復ぶん**を実装する。往復上限・停止・レビュー/対話のモード切替は PR 2。

## 何をするか

セルの宛先メニューの `⇄` を押すと:

1. 自分の直前ターンを相手セルに送って**自動送信**
2. 相手のターン完了を待つ
3. 相手の返答を自分に送って自動送信
4. 自分のターン完了を待って終了

## 設計判断

### 完了判定は「新しいターンが現れたこと」

当初は活動フラグの `done` を待つつもりだったが、**ユーザーが見ているペインでは `Stop` が `waiting` を立てない**（`activityHookEffects` が意図的に抑制する）。実機で確認済み。`done` を待つと見ているセルで永久に待つ。

代わりに `/api/transcript/last-turn` をポーリングし、**(prompt, reply) の組が変化したか**で判定する。送ったテキストが相手の次の prompt になるので、返答の文言が偶然同じでも必ず変化が観測できる。

これは同時に**二重送信ガード**でもある。「以前は存在しなかったターン」が記録されるまで次を送らないので、遅いエージェントには待つだけで、二度送ることがない。

### 承認待ちの検知は不要

ループは**ターン完了を観測してからしか送らない**。承認プロンプトで止まっているセッションはターンを完了しないので、送信対象にならない。待ってタイムアウトし、表面化するだけ＝安全側に倒れる。

### 戻し側は返答だけを送る

A が送ったテキストは B の `user_message` になる。そのまま戻すと **A は自分の文章を受け取る**（往復ごとに）。`/api/transcript/last-turn?as=reply` を追加し、戻し側は prompt ブロックを落として「相手からの返答」として枠づける。

### 自動送信は bracketed paste ＋ 遅延 CR

同一バーストの text+CR は claude の TUI がペースト扱いして CR を飲む。bracketed paste に CR を貼り付けても、ペーストが確定する前に CR が届いて落ちる。サーバの `attachDraftInjection` が既に踏んでいる地雷なので同じ形にする。

## 既知の制約

- **起動直後のセルに送ると入力が失われる**。codex の TUI が立ち上がりきる前のペーストは捨てられる（実測で確認）。実運用では「直前ターンを持つセル」を相手に選ぶので通常は起きないが、一度も使っていないセルを相手にすると待ってタイムアウトする
